①What you'll end up with

One private repository on GitHub becomes the single source of truth for your vault. Every device you own holds a full copy of your notes and quietly syncs with that repo:

  • Edit anywhere — your laptop, a desktop, your phone — and the change flows to the others.
  • Full history — every save is a version you can roll back to. Nothing is ever truly lost.
  • Works offline — each device has the real files locally; it syncs when it's back online.
  • You own the data — plain Markdown files in your own GitHub account. No lock-in.
  • Free — for the computer-to-computer part. (Phone has a free path and a paid easy-button — both below.)
Your goal Open the same notes on your phone and on a second computer that you have on your main one. That's exactly what this delivers. The trick is just: one repo, every device talks to it.

②The core idea (how Jon does it)

Three pieces, that's the whole thing:

  • Obsidian — the notes app. Free. Your vault is just a folder of Markdown files on disk.
  • The Obsidian Git plugin — a community plugin that automatically commits your changes and pushes/pulls them to GitHub on a timer. Set it once and forget it.
  • A private GitHub repo — the central hub every device syncs with.

Think of GitHub as the hub of a bike wheel and each device as a spoke. A device pulls the latest notes when it starts, you edit, and it commits + pushes your changes back every few minutes. The next device pulls them down. That's it.

Why Git instead of Dropbox/iCloud? Because you get versioned history (undo anything, see what changed when), it handles "edited in two places" gracefully, and your notes stay as portable plain text you fully control.

③Step 1 — Create your GitHub repo

  1. Make a free account at github.com if you don't have one.
  2. Create a new repository — name it something like my-vault, set it to Private, and leave it empty (no README).
  3. You'll need a way to authenticate. Two easy options:
    • Personal Access Token (PAT) — GitHub → Settings → Developer settings → Personal access tokens. Create one with repo scope. You'll paste this into the Obsidian Git plugin (it acts as your password). This is what Jon uses.
    • GitHub CLI — install gh and run gh auth login once per computer. It stores the credential for you so Obsidian never has to ask.
Tip A token gives whoever holds it access to your repo — treat it like a password. Don't paste it into notes that themselves get synced.

④Step 2 — Set up your MAIN computer

This is the device whose notes you want everywhere. macOSWindows

Install the pieces

  1. Install Git.
    • Mac: it's usually already there. If not, install Homebrew then brew install git, or just run git --version and macOS will offer to install it.
    • Windows: download Git for Windows and install with defaults.
  2. Install Obsidian from obsidian.md and open your existing vault (or make a new one).
  3. Install the Obsidian Git plugin: Obsidian → Settings → Community plugins → Browse → search "Git" (by Vinzent) → Install → Enable.

Connect the vault to your repo

Open a terminal in your vault folder and link it to the repo you made (this also pushes your notes up the first time):

cd /path/to/your/vault
git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/YOUR-USERNAME/my-vault.git
git push -u origin main

If you used a PAT, Git will ask for a username + password on that first push — paste the token as the password.

Turn on automatic sync

In Settings → Community plugins → Git, set these (this mirrors Jon's setup):

  • Auto-backup (commit + push) every 10 minutes — saves and uploads your changes on a timer.
  • Pull updates on startup: ON — grabs the latest from your other devices when Obsidian opens.
  • Auto-pull every 10 minutes — keeps you current while you work.
  • Pull before push: ON — avoids most conflicts automatically.
  • Commit message: vault backup: {{date}} — tidy, timestamped history.
That's the engine Once this is on, you never think about it again. Edit notes, and every ~10 minutes they're safely on GitHub and on their way to your other devices.

⑤Step 3 — Add a second computer

Same idea, but instead of pushing an existing vault up, you pull the repo down. macOSWindows

  1. Install Git and Obsidian (as above).
  2. Clone your repo to a folder: 
    git clone https://github.com/YOUR-USERNAME/my-vault.git
  3. In Obsidian, choose "Open folder as vault" and pick the folder you just cloned.
  4. Install + enable the Obsidian Git plugin and apply the same settings as Step 2.

Done — this computer now auto-pulls others' changes and pushes its own. Edits flow both ways automatically.

⑥Step 4 — Your phone (pick one path)

The phone is the one spot with a real choice. Pick whichever fits how much fiddling you'll tolerate.

What it is Obsidian's own official sync service. Flip it on, sign in on each device, done. The same Git approach as your computers, extended to the phone.

Path A — Obsidian Sync (easiest)

  1. In Obsidian on your main computer: Settings → Sync → set up a remote vault, sign into an Obsidian account, turn Sync on.
  2. Install the Obsidian app on your phone, sign in, connect to the same remote vault.

Note: you can run Obsidian Sync and keep the free Git backup on your computers at the same time — Sync for live device-to-device, Git for versioned backup. Many people do exactly that.

Path B — Free, Git-based

  • Android: install Obsidian, clone or open the vault, add the Obsidian Git plugin, apply the same settings as your computer. That's it.
  • iPhone: install Working Copy (a Git client for iOS) → clone your repo into it → install the Obsidian app → "Open folder as vault" pointed at the Working Copy folder. Pull when you start, push when you finish (Working Copy can do this with a Shortcut so it's one tap).
iPhone reality check The Obsidian Git plugin is unreliable on iPhone for anything but tiny vaults — that's why Working Copy (or Obsidian Sync) is the real answer there. Android has no such issue.

⑦The .gitignore you want

A .gitignore file in your vault tells Git which files not to sync. You want to skip per-device UI state and anything secret, while keeping your notes and your plugin setup (so a new device comes up already configured). Drop this in the root of your vault:

# Per-device Obsidian UI state (don't sync — causes noise/conflicts)
.obsidian/workspace.json
.obsidian/workspace-mobile.json

# OS junk
.DS_Store
Thumbs.db

# Temp / backup files
*.tmp
*.bak

# Never sync secrets (API keys, tokens, .env files)
.env
*.secret

Keep the rest of .obsidian/ in the repo — that's how your community plugins and settings travel to every device automatically.

⑧Good habits & gotchas

  • Let it land before you edit elsewhere. When you switch devices, give it a moment to pull the latest (or hit the pull button) before you start typing. Editing the same note on two devices that haven't synced is the main way to create a conflict.
  • Push before you close. On the phone especially, do a manual push when you finish so the change is up before you lock the screen.
  • Conflicts are rare and survivable. If two devices changed the same note, Git inserts both versions with <<<< / >>>> markers. Open the note, keep the text you want, delete the markers, save. Nothing is lost — both versions are right there.
  • Watch attachments/size on mobile. Huge image/PDF vaults sync slowly on phones. If yours is media-heavy, Obsidian Sync (Path A) handles it more gracefully than Git on a phone.

⑨Troubleshooting — the snags I actually hit

Straight talk Setup is mostly smooth, but a handful of things cost me real time getting this running. Here they are with the fixes, so you skip the pain.

1. "Authentication failed" when the plugin tries to push (macOS)

The Obsidian Git plugin can't always show a login box, so it just silently fails to authenticate. Two reliable fixes — easiest first:

  • Use the GitHub CLI. Install gh, then run gh auth login followed by gh auth setup-git. Git now authenticates automatically and the plugin stops asking.
  • Or store a token in your OS keychain (macOS Keychain / Windows Credential Manager) so HTTPS pushes work without prompts.

Power-user route (what I ended up on): point Git's GIT_ASKPASS at a tiny helper script so the plugin can feed the token non-interactively. Only worth it if the above two don't stick.

2. A freshly-cloned vault on a SECOND computer or phone can't push you'll hit this

A plain git clone over HTTPS has no saved credentials, so the very first push fails. On that new device, either run gh auth login + gh auth setup-git, or do one push with the helper wired in:

git -c credential.helper='!gh auth git-credential' push origin main

After that first authenticated push, it remembers. This is the single most common wall when adding a device.

3. You accidentally committed a secret

If a token, .env, or a config file with API keys ever gets committed, deleting the file later does NOT remove it from history — it still lives in old commits. So:

  • Get your .gitignore right before your first commit (see section ⑦).
  • If it already happened: rotate the leaked key (assume it's burned) and scrub history with git filter-repo. Don't lean on "but the repo is private."

4. Windows: every file shows as "changed," or phantom conflicts appear

That's line-ending churn (Windows CRLF vs. Mac/Linux LF). Set it once per Windows machine:

git config --global core.autocrlf input

5. "Updates were rejected" / push fails after editing on two devices

Both devices committed and now disagree. Keep "pull before push" ON (it's in the recommended settings) and it usually self-heals. If a specific note conflicts, open it — you'll see <<<<<<< / >>>>>>> markers. Keep the text you want, delete the markers, save, and it pushes clean.

6. iPhone + Obsidian Git plugin = flaky

Don't fight it. On iOS use Working Copy or Obsidian Sync (Step ⑥) instead of the plugin.

⑩Jon's actual config (copy this)

For reference, here are the real Obsidian Git settings Jon runs — translated to the underlying values, in case you want to match them exactly:

Auto-backup (commit + push)   every 10 minutes
Auto-pull                     every 10 minutes
Pull on startup               ON
Pull before push              ON
Sync method                   merge
Commit message                "vault backup: {{date}}"
Date format                   YYYY-MM-DD HH:mm:ss
Authentication                GitHub Personal Access Token (stored in the OS keychain)
Repo                          one private GitHub repo, branch: main

The history ends up looking like a clean stream of vault backup: 2026-06-02 16:27:18 commits — every save timestamped and recoverable.

⑪Quick-start checklist

  1. Create a free private GitHub repo.
  2. On your main computer: install Git + Obsidian + the Obsidian Git plugin.
  3. Link the vault to the repo and do the first push.
  4. Turn on auto-backup + auto-pull (10 min) and pull on startup.
  5. On a second computer: git clone the repo, open as vault, same plugin + settings.
  6. On your phone: pick Obsidian Sync (easy, paid) or Working Copy (iPhone) / Obsidian Git (Android) (free).
  7. Add the .gitignore. Done — edit anywhere, it syncs everywhere.
